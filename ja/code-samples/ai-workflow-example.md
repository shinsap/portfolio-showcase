# AI支援開発ワークフロー例

公開用に簡略化した、AIツールを使った開発の進め方です。

## 1. 目的を定義する

最初にプロダクトレベルで整理します。

- 誰が使うのか
- 何を解決するのか
- どんなデータを扱うのか
- 公開してはいけないものは何か
- デプロイや運用はどうするのか

## 2. AIに初期案を出させる

AIには主に以下を依頼します。

- コンポーネント構成案
- データモデル案
- edge caseの洗い出し
- 実装パターンの比較

## 3. 実装前にレビューする

AIの出力はそのまま採用せず、以下を確認します。

- 既存アプリの設計と合っているか
- 不要な抽象化が増えていないか
- secretの扱いに問題がないか
- 外部サービス前提が正しいか
- testやdeploymentへの影響は何か

## 4. 小さく実装する

変更は小さく区切ります。

- 1 feature / 1 fix単位
- 必要な範囲のtest
- 運用が変わる場合はREADMEやhandover noteを更新

## 5. 運用面を確認する

active repoでは以下を確認します。

- GitHub Actions
- secrets names
- webhooks / deploy keys
- stale deployment automation
- old repositoriesのarchive

## 結果

AIは実装速度を上げますが、プロダクト適合性、セキュリティ、deployment、保守性の最終責任は開発者が持つ、という方針です。
